## Listening to Events

<Tabs syncKey="framework">
  <TabItem label="Vanilla JS">
```typescript
import { queryGrid } from '@toolbox-web/grid';
import type { CellClickDetail, CellCommitDetail } from '@toolbox-web/grid';

const grid = queryGrid<Employee>('#my-grid');

// Type-safe event listening
grid.addEventListener('cell-click', (e: CustomEvent<CellClickDetail<Employee>>) => {
  console.log(e.detail.row, e.detail.field);
});

// Cancelable events — call preventDefault() to reject the action
grid.addEventListener('cell-commit', (e: CustomEvent<CellCommitDetail<Employee>>) => {
  if (!isValid(e.detail.value)) {
    e.preventDefault(); // Rejects the edit
  }
});
```
  </TabItem>
  <TabItem label="React">
```tsx
import { DataGrid, useGrid, useGridEvent } from '@toolbox-web/grid-react';
import type { CellClickDetail, CellCommitDetail } from '@toolbox-web/grid';

function EmployeeGrid() {
  const gridRef = useGrid<Employee>();

  // Type-safe event hook
  useGridEvent(gridRef, 'cell-click', (e) => {
    console.log(e.detail.row, e.detail.field);
  });

  // Or use event handler props
  const handleCellCommit = useCallback((detail: CellCommitDetail<Employee>) => {
    if (!isValid(detail.value)) {
      // Cancelable — return false or call e.preventDefault()
    }
  }, []);

  return <DataGrid ref={gridRef} rows={employees} onCellCommit={handleCellCommit} />;
}
```

## Cancelable Events

Four events support `preventDefault()` to reject the action:

| Event | Plugin | What Canceling Does |
|-------|--------|-------------------|
| `cell-commit` | Editing | Reverts the cell to its previous value |
| `row-commit` | Editing | Reverts the entire row to pre-edit state |
| `column-move` | Reorder | Rejects the column drag-and-drop |
| `row-move` | Row Reorder | Rejects the row drag/keyboard move |

## Core Events

These events are always available — no plugins required.

| Event | Detail Type | Description |
|-------|------------|-------------|
| `cell-click` | [`CellClickDetail<TRow>`](/grid/api/core/interfaces/cellclickdetail/) | User clicks a cell. Provides row data, cell position, and original mouse event. |
| `cell-activate` | [`CellActivateDetail<TRow>`](/grid/api/core/interfaces/cellactivatedetail/) | Cell activated via keyboard (Enter) or pointer. Includes trigger type. |
| `cell-change` | [`CellChangeDetail<TRow>`](/grid/api/core/interfaces/cellchangedetail/) | Data mutation on a cell. Includes old/new values, field name, and all changed rows. |
| `row-click` | [`RowClickDetail<TRow>`](/grid/api/core/interfaces/rowclickdetail/) | User clicks a row. Provides row data, index, and original mouse event. |
| `sort-change` | [`SortChangeDetail`](/grid/api/core/interfaces/sortchangedetail/) | Column sort order changes. Direction: `1` (asc), `-1` (desc), `0` (cleared). |
| `column-resize` | [`ColumnResizeDetail`](/grid/api/core/interfaces/columnresizedetail/) | User finishes resizing a column. Provides field name and new width. |
| `column-state-change` | [`GridColumnState`](/grid/api/core/interfaces/gridcolumnstate/) | Column visibility, width, or order changes. Complete state snapshot. |

:::note
`activate-cell` is deprecated — use `cell-activate` instead.
:::

## Plugin Events

### Editing

Requires `EditingPlugin`. Import from `@toolbox-web/grid/plugins/editing`.

| Event | Detail Type | Cancelable | Description |
|-------|------------|-----------|-------------|
| `cell-commit` | [`CellCommitDetail<TRow>`](/grid/plugins/editing/interfaces/cellcommitdetail/) | ✅ | Cell value committed (Enter/blur). Cancel to reject. |
| `row-commit` | [`RowCommitDetail<TRow>`](/grid/plugins/editing/interfaces/rowcommitdetail/) | ✅ | Focus leaves row in row-edit mode. Cancel to revert. |
| `cell-cancel` | [`CellCancelDetail`](/grid/plugins/editing/interfaces/cellcanceldetail/) | — | User presses Escape; cell reverts to pre-edit value. |
| `edit-open` | [`EditOpenDetail<TRow>`](/grid/plugins/editing/interfaces/editopendetail/) | — | Row editing begins (row-edit mode only). |
| `before-edit-close` | [`BeforeEditCloseDetail<TRow>`](/grid/plugins/editing/interfaces/beforeeditclosedetail/) | — | Synchronous, just before editor DOM is destroyed. |
| `edit-close` | [`EditCloseDetail<TRow>`](/grid/plugins/editing/interfaces/editclosedetail/) | — | Row editing ends. Includes `reverted` flag. |
| `changed-rows-reset` | [`ChangedRowsResetDetail<TRow>`](/grid/plugins/editing/interfaces/changedrowsresetdetail/) | — | `resetChangedRows()` called programmatically. |
| `dirty-change` | [`DirtyChangeDetail<TRow>`](/grid/plugins/editing/interfaces/dirtychangedetail/) | — | Dirty tracking state changes. Requires `dirtyTracking: true`. |
| `baselines-captured` | [`BaselinesCapturedDetail`](/grid/plugins/editing/interfaces/baselinescaptureddetail/) | — | New baseline snapshots captured post-render. |
